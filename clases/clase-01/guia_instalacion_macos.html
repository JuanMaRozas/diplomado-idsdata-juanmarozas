¿Qué vamos a instalar y para qué?
El mapa completo antes de empezar

Vamos a instalar 4 herramientas que trabajan juntas. Necesitas todas:

⚠️
Importante: Este curso usa Python 3.12. Las versiones 3.13 y 3.14 tienen incompatibilidades con algunas librerías del curso. Instala exactamente la 3.12.
💡
¿Cómo abrir la Terminal en Mac? Presiona Cmd + Espacio, escribe Terminal y presiona Enter. La Terminal es la ventana de texto negra donde escribirás comandos.
2
Python 3.12
El lenguaje de programación del curso

¿Qué es Python?

Python es el lenguaje de programación que usaremos todo el curso. Es como el "motor" que entiende y ejecuta tu código. Sin Python instalado, nada funciona. Piénsalo como instalar el motor de un auto antes de poder manejar.

1

Primero verifica si ya tienes Python 3.12. Abre la Terminal y escribe:

python3 --version

Si muestra Python 3.12.x puedes saltarte esta sección. Si muestra 3.13, 3.14 o nada, sigue los pasos.

2

Ve a la página oficial:

🔗 python.org/downloads/macos

Busca la sección "Python 3.12". Descarga el archivo "macOS 64-bit universal2 installer" (.pkg).

3

Abre el archivo .pkg descargado y sigue el asistente haciendo clic en "Continuar" en cada paso hasta completar la instalación.

💡
Si Mac te pide permiso para instalar, introduce tu contraseña de usuario. Es normal y necesario.
4

Instala los certificados SSL necesarios. Al terminar la instalación, abre la carpeta de Python 3.12 en Finder y haz doble clic en "Install Certificates.command".

Alternativamente, desde la Terminal:

/Applications/Python\ 3.12/Install\ Certificates.command
✅ Verificación — en la Terminal escribe:
python3 --version
Debe mostrar: Python 3.12.x
3
VS Code
Tu editor de código

¿Qué es VS Code?

VS Code es donde escribirás y ejecutarás tu código. Tiene un editor, una terminal integrada y se conecta directamente con Python y Git. Piénsalo como el Word del código, pero con superpoderes.

1

Descarga VS Code:

🔗 code.visualstudio.com

Haz clic en "Download for Mac". Descargará un archivo .zip.

2

Abre el .zip descargado — aparecerá el ícono de VS Code. Arrastra el ícono de VS Code a tu carpeta Aplicaciones.

3

Abre VS Code desde Aplicaciones. La primera vez macOS puede preguntarte si confías — haz clic en "Abrir".

4

Instala el comando code en la Terminal para poder abrir proyectos con un solo comando:

Presiona Cmd + Shift + P → escribe Shell Command: Install 'code' command in PATH → Enter.

✅
Después de esto podrás escribir code . en la Terminal para abrir cualquier carpeta en VS Code.
5

Instala las extensiones esenciales con Cmd + Shift + X:

Busca e instala: Python (Microsoft), Data Wrangler (Microsoft), GitHub Copilot (GitHub), GitLens (GitKraken), Rainbow CSV.

4
Git
Control de versiones

¿Qué es Git?

Git registra cada cambio en el código y lo sincroniza con GitHub. Piénsalo como el historial de versiones de un documento Word, pero para código y mucho más poderoso. Te permite traer el material nuevo del curso con un solo comando.

1

Verifica si ya tienes Git. Abre la Terminal y escribe:

git --version

Si aparece un número de versión, ya está instalado. Si macOS te ofrece instalar las Command Line Tools, acepta — incluye Git.

2

Si Git no está instalado, la forma más simple es instalarlo a través de Homebrew. Primero instala Homebrew (si no lo tienes):

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Luego instala Git:

brew install git
3

Configura tu identidad (usa el mismo correo de GitHub):

git config --global user.name "Tu Nombre" git config --global user.email "tucorreo@gmail.com"
✅ Verificación:
git --version
Debe mostrar: git version 2.x.x
5
GitHub — tu cuenta en la nube
Donde vive el código del curso

¿Qué es GitHub?

GitHub es donde el profe guarda todo el material del curso. Piénsalo como Google Drive para código. Tú solo necesitas descargar el material — no tienes que subir nada.

1

Ve a 🔗 github.com → Sign up.

2

Crea tu cuenta con un correo que uses. Elige un nombre de usuario profesional — será parte de tu portafolio.

3

Confirma tu correo haciendo clic en el enlace que GitHub te envió.

6
Clonar el repositorio del curso
Descargar el material a tu Mac

¿Qué es un repositorio?

Un repositorio es una carpeta que contiene el código y todo su historial. "Clonar" significa descargar una copia completa a tu computador. Después, con git pull traes los cambios nuevos del profe antes de cada clase.

1

Abre la Terminal (Cmd + Espacio → escribe Terminal).

2

Ve a tu carpeta personal y clona el repo:

# Ir a la carpeta personal cd ~ # Descargar el repo del curso git clone https://github.com/JuanMaRozas/diplomado-idsdata-juanmarozas.git # Entrar a la carpeta cd diplomado-idsdata-juanmarozas
3

Abre la carpeta en VS Code:

code .

VS Code se abrirá con toda la estructura del curso. Si pregunta si confías en la carpeta, haz clic en "Sí, confío".

💡
Antes de cada clase, trae el material nuevo:
cd ~/diplomado-idsdata-juanmarozas git pull
7
El entorno virtual
Tu caja de herramientas del curso

¿Qué es un entorno virtual?

Un entorno virtual (.venv) es una carpeta aislada con las librerías específicas de este curso. Imagina una caja de herramientas exclusiva: tiene exactamente lo que necesitas (pandas, boto3, etc.), en las versiones correctas, sin interferir con otros proyectos de tu Mac.

1

En la Terminal, asegúrate de estar en la raíz del repo:

cd ~/diplomado-idsdata-juanmarozas
2

Ejecuta el script de instalación:

bash clases/clase-01/material/setup_local.sh

El script tomará 3–5 minutos instalando las librerías. Al finalizar verás:

OK -> pandas 2.2.2 ================================================ Entorno listo en: /Users/TuNombre/diplomado-idsdata-juanmarozas/.venv Activa con: Mac/Linux -> source .venv/bin/activate ================================================
💡
¿Tienes otras versiones de Python instaladas? No hay problema — el script busca específicamente la 3.12 y crea el entorno con ella, sin tocar las otras versiones.
⚠️
Si ves "ERROR: No se encontró Python 3.12": necesitas instalarlo. Descárgalo desde python.org/downloads — no desinstales lo que ya tienes, solo agrega la 3.12.
3

Activa el entorno virtual:

source .venv/bin/activate

Cuando está activo, verás (.venv) al inicio de la línea:

(.venv) JuanMa@Mac diplomado-idsdata-juanmarozas %
✅
El (.venv) indica que el entorno está activo. Actívalo siempre antes de trabajar con el código del curso.
💡
Rutina antes de cada clase:
cd ~/diplomado-idsdata-juanmarozas git pull source .venv/bin/activate
8
Seleccionar el intérprete en VS Code
Decirle a VS Code qué Python usar

VS Code necesita saber que debe usar el Python del .venv del curso, no el Python del sistema.

1

Con la carpeta del repo abierta en VS Code, presiona Cmd + Shift + P.

2

Escribe Python: Select Interpreter y presiona Enter.

3

Elige la opción que contenga .venv:

Elige '.venv': venv. Verás el intérprete activo en la barra inferior de VS Code.

⚠️
Si .venv no aparece: el entorno no se creó correctamente. Vuelve a la sección 7 y corre el setup_local.sh de nuevo.
9
Verificación final
Confirma que todo funciona

Con el .venv activo en la Terminal, ejecuta:

python3 -c "import pandas, numpy, boto3, sweetviz; print('✅ Todo OK ->', pandas.__version__)"
✅
Si ves: ✅ Todo OK -> 2.2.2 — tu entorno está perfectamente configurado.
❌
Si ves "ModuleNotFoundError": el entorno no está activo. Corre source .venv/bin/activate y vuelve a intentarlo.

🎉 ¡Listo! Tu entorno macOS está completamente configurado.

Antes de cada clase: cd ~/diplomado-idsdata-juanmarozas && git pull && source .venv/bin/activate
