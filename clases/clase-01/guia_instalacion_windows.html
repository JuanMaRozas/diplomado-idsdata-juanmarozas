Python 3.12
El lenguaje de programación del curso

¿Qué es Python?

Python es el lenguaje de programación que usaremos todo el curso. Piénsalo como el "motor" que entiende y ejecuta tu código. Sin Python instalado, nada funciona. Es como instalar el motor de un auto antes de poder manejarlo.

1

Abre tu navegador y ve a la página oficial de Python:

🔗 python.org/downloads/windows

Busca la sección "Python 3.12". Fíjate bien — no descargues la versión más reciente si es 3.13 o superior.

2

Descarga el instalador de 64 bits: busca el enlace que diga "Windows installer (64-bit)" para Python 3.12.x

3

Ejecuta el instalador que descargaste. Aparecerá una ventana. Antes de hacer clic en "Install Now", marca estas dos casillas:

⚠️
"Add Python 3.12 to PATH" es la casilla más importante. Si no la marcas, tu computador no encontrará Python y nada funcionará. Asegúrate de que esté marcada antes de continuar.
4

Haz clic en "Install Now" y espera que termine. Puede tomar 2–3 minutos.

Al finalizar, haz clic en "Close".

✅ Verificación — abre CMD (busca "cmd" en el menú inicio) y escribe:
python --version
Debe mostrar: Python 3.12.x
⚠️
Si dice "Python 3.13" o "3.14": desinstálalo desde Configuración → Aplicaciones y vuelve a instalar la 3.12 siguiendo el paso 1.
3
VS Code
Tu editor de código

¿Qué es VS Code?

VS Code es el programa donde escribirás tu código, lo correrás y verás los resultados. Piénsalo como el Word del código — pero mucho más poderoso. Tiene un editor, una terminal integrada y se conecta directamente con Python y Git.

1

Descarga VS Code desde la página oficial:

🔗 code.visualstudio.com

Haz clic en el botón azul grande "Download for Windows".

2

Ejecuta el instalador. En el paso "Seleccionar tareas adicionales", marca estas opciones:

3

Haz clic en "Siguiente" e "Instalar". Cuando termine, abre VS Code.

4

Instala las extensiones esenciales. En VS Code, haz clic en el ícono de extensiones en la barra izquierda (el que parece 4 cuadrados) o presiona Ctrl + Shift + X.

Busca e instala estas extensiones una por una:

Instala: Python (Microsoft), Data Wrangler (Microsoft), GitHub Copilot (GitHub), GitLens (GitKraken), Rainbow CSV.

5

Cambia la terminal de VS Code a Git Bash (esto es muy importante en Windows):

Presiona Ctrl + Shift + P → escribe Terminal: Select Default Profile → elige Git Bash.

💡
Si Git Bash no aparece en la lista, primero instala Git (sección 4 de esta guía) y luego vuelve a este paso.
4
Git y Git Bash
La terminal y el control de versiones

¿Qué es Git?

Git es la herramienta que registra cada cambio en tu código y lo sincroniza con GitHub (la nube). Piénsalo como el historial de cambios de un documento de Word, pero para código y mucho más poderoso. Git Bash es una terminal (ventana de comandos) que viene con Git y funciona igual que en Mac — es la que usaremos en este curso.

⚠️
En Windows hay varias terminales (PowerShell, CMD, Git Bash). Para este curso usa SIEMPRE Git Bash. Los comandos del curso no funcionan en PowerShell ni en CMD.
1

Descarga Git para Windows:

🔗 git-scm.com/download/win

La descarga comenzará automáticamente. Si no, haz clic en el enlace de "64-bit Git for Windows Setup".

2

Ejecuta el instalador. La mayoría de las opciones pueden dejarse por defecto. Solo presta atención a esta pantalla:

Selecciona "Git from the command line and also from 3rd-party software". Todo lo demás déjalo por defecto.

3

Completa la instalación haciendo clic en "Next" en cada pantalla y finalmente "Install".

Cuando termine, cierra y vuelve a abrir VS Code para que detecte Git.

4

Configura tu identidad en Git. Abre Git Bash (búscalo en el menú inicio) y escribe estos dos comandos, reemplazando con tu nombre y correo:

git config --global user.name "Tu Nombre Aquí" git config --global user.email "tucorreo@gmail.com"

Esto le pone tu "firma" a cada cambio que hagas en el código.

✅ Verificación — en Git Bash escribe:
git --version
Debe mostrar: git version 2.4x.x.windows.x

GitHub — tu cuenta en la nube
Donde vive el código del curso

¿Qué es GitHub?

GitHub es el servicio en la nube donde el profe guarda todo el material del curso (scripts, guías, presentaciones). Piénsalo como Google Drive, pero para código. Tú vas a descargar el material desde ahí antes de cada clase. No necesitas subir nada — solo descargar.

1

Ve a 🔗 github.com y haz clic en "Sign up".

2

Ingresa tu correo electrónico, crea una contraseña y elige un nombre de usuario. Elige un nombre profesional — será parte de tu portafolio técnico.

3

Confirma tu correo haciendo clic en el enlace que te enviará GitHub.

Clonar el repositorio del curso
Descargar el material a tu computador

¿Qué es un repositorio?

Un repositorio (o "repo") es una carpeta especial que guarda el código y todo su historial de cambios. "Clonar" significa descargar una copia completa de esa carpeta a tu computador. Una vez clonado, puedes traer los cambios nuevos del profe con un solo comando: git pull.

Clonas una vez. Después solo haces git pull para traer lo nuevo.
1

Abre Git Bash desde el menú inicio.

⚠️
Usa Git Bash, no PowerShell ni CMD. Si usas PowerShell los comandos no funcionarán.
2

Ve a tu carpeta personal y clona el repositorio:

# Ir a la carpeta personal cd ~ # Descargar el repo del curso git clone https://github.com/JuanMaRozas/diplomado-idsdata-juanmarozas.git # Entrar a la carpeta cd diplomado-idsdata-juanmarozas
3

Abre la carpeta en VS Code:

code .

VS Code se abre con toda la estructura del curso visible. Si te pregunta si confías en la carpeta, haz clic en "Sí, confío".

7
El entorno virtual
Tu caja de herramientas del curso

¿Qué es un entorno virtual?

Un entorno virtual (.venv) es una carpeta aislada donde instalas las librerías de este proyecto sin afectar el resto del computador. Imagina que es una caja de herramientas exclusiva para el curso: tiene exactamente las herramientas que necesitas, en las versiones correctas, y no interfiere con otros proyectos.

1

En Git Bash, asegúrate de estar en la raíz del repo:

cd ~/diplomado-idsdata-juanmarozas
2

Ejecuta el script de instalación:

bash clases/clase-01/material/setup_local.sh

El script tomará 3–5 minutos. Verás texto descargando. Si todo sale bien, al final aparecerá:

OK -> pandas 2.2.2 ================================================ Entorno listo en: /c/Users/TuNombre/diplomado-idsdata-juanmarozas/.venv Activa con: Windows -> source .venv/Scripts/activate ================================================
💡
¿Tienes otras versiones de Python instaladas? No hay problema — el script busca específicamente la 3.12 y crea el entorno con ella, sin tocar las otras versiones.
⚠️
Si ves "ERROR: No se encontró Python 3.12": necesitas instalarlo. Descárgalo desde python.org/downloads — no desinstales lo que ya tienes, solo agrega la 3.12.
3

Activa el entorno virtual:

source .venv/Scripts/activate

Cuando está activo, verás (.venv) al inicio de la línea:

(.venv) TuNombre@PC ~/diplomado-idsdata-juanmarozas $
✅
El (.venv) al inicio indica que el entorno está activo. Siempre actívalo antes de trabajar con el código del curso.
💡
Rutina antes de cada clase:
cd ~/diplomado-idsdata-juanmarozas git pull source .venv/Scripts/activate
8
Seleccionar el intérprete en VS Code
Decirle a VS Code qué Python usar

VS Code necesita saber que debe usar el Python del .venv del curso, no el Python global del sistema.

1

Con la carpeta del repo abierta en VS Code, presiona Ctrl + Shift + P.

2

Escribe Python: Select Interpreter y presiona Enter.

3

Aparece una lista de intérpretes disponibles. Elige el que contenga .venv:

Elige la opción que diga '.venv': venv o simplemente .venv.

⚠️
Si .venv no aparece en la lista: el entorno virtual no se creó correctamente. Vuelve a la sección 7 y corre el setup_local.sh de nuevo.
9
Verificación final
Confirma que todo funciona

Con el .venv activo en Git Bash, corre este comando:

python -c "import pandas, numpy, boto3, sweetviz; print('✅ Todo OK ->', pandas.__version__)"
✅
Si ves: ✅ Todo OK -> 2.2.2 — tu entorno está perfectamente configurado y estás listo para la clase.
❌
Si ves "ModuleNotFoundError": el intérprete activo no es el .venv. Asegúrate de haber activado el entorno con source .venv/Scripts/activate y de haber seleccionado el intérprete .venv en VS Code.

🎉 ¡Listo! Tu entorno Windows está completamente configurado.

Antes de cada clase solo necesitas correr: cd ~/diplomado-idsdata-juanmarozas && git pull && source .venv/Scripts/activate
