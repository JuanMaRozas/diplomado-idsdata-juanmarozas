El entorno de trabajo
Python, librerías, entorno virtual y dependencias

¿Qué es el "entorno"?

El entorno es el conjunto de herramientas con las que corre tu código: la versión de Python, las librerías instaladas y sus versiones exactas. Un entorno bien definido garantiza que tu código funcione igual en tu máquina, en la del compañero y en la nube.

Librerías y dependencias

Python viene con herramientas básicas, pero para trabajar con datos necesitamos librerías adicionales — código escrito por otros que extiende las capacidades del lenguaje. Las que usamos en el curso:

El entorno virtual (.venv)

Un entorno virtual es una carpeta aislada donde se instalan las librerías de un proyecto específico, sin afectar otros proyectos ni el sistema. Cada proyecto tiene su propio entorno con sus propias versiones.

Imports
Cómo traer librerías y funciones al script

Un import le dice a Python: "necesito usar esta librería". Sin importarla, no puedes usarla, aunque esté instalada. Los imports van siempre al inicio del archivo.

import … as (alias)

Cuando la librería tiene un nombre largo o hay una convención universal, se le pone un alias más corto con as. El alias es el nombre que usas en el resto del código.

PYTHON01_leer_csv.py — línea 17
import pandas as pd

# A partir de aquí, en vez de escribir "pandas.read_csv(...)"
# escribes "pd.read_csv(...)"  ← mucho más corto
pagos = pd.read_csv("archivo.csv")
Convención del mundoLo que significa
import pandas as pdpandas se llama pd en todo el código
import numpy as npnumpy se llama np
import matplotlib.pyplot as pltpyplot se llama plt
💡
Son convenciones universales. Si escribes import pandas as datos funciona, pero confundirás a cualquier persona que lea tu código. Sigue siempre las convenciones.

from … import (importar algo específico)

En vez de importar toda la librería, puedes traer solo lo que necesitas. Esto es más eficiente y el código queda más limpio.

PYTHON01_leer_csv.py — línea 15 | 02_glob_inventario.py — líneas 13-14
from pathlib import Path
# Ahora puedes usar directamente "Path(...)" en vez de "pathlib.Path(...)"

from glob import glob
# Ahora puedes usar directamente "glob(...)" en vez de "glob.glob(...)"
📌
Regla práctica: usa import X as alias para librerías grandes (pandas, numpy). Usa from X import Y cuando solo necesitas una función específica de una librería.

Variables y constantes
Guardar valores para usarlos después

Una variable es un nombre que guarda un valor en memoria. En Python no necesitas declarar el tipo — Python lo infiere automáticamente.

PYTHON
# Asignar un valor a una variable
nombre  = "Juan"         # str (texto)
edad    = 35             # int (entero)
precio  = 99.9           # float (decimal)
activo  = True           # bool (verdadero/falso)

# Reasignar (Python permite cambiar el valor)
precio = 120.0           # ahora precio vale 120.0

Las constantes son variables que por convención no se modifican. Se escriben en MAYÚSCULAS.

Convención de nombres: variables en minusculas_con_guion, constantes en MAYUSCULAS, clases en PalabrasJuntas. No es obligatorio pero es el estándar universal de Python (PEP 8).

Definir funciones (def)
Empaquetar código reutilizable

¿Para qué sirve una función?

Una función es un bloque de código con nombre que puedes ejecutar ("llamar") cuando quieras. En vez de repetir 10 líneas cada vez que inspeccionas un DataFrame, las empaquetas en una función y la llamas con una sola línea. Regla de oro: si copias y pegas el mismo código más de dos veces, ahí va una función.

PYTHONAnatomía de una función
def nombre_de_la_funcion(parametro1, parametro2):
    """Docstring: describe qué hace la función."""
    # Cuerpo de la función (indentado con 4 espacios)
    resultado = parametro1 + parametro2
    return resultado    # valor que devuelve

Parámetros y type hints

Los parámetros son los datos que recibe la función. Los type hints (anotaciones de tipo) documentan qué tipo de dato se espera — Python no los exige, pero hacen el código mucho más legible y Copilot los usa para dar mejores sugerencias.

Docstrings

El docstring es la descripción de la función. Va entre triple comillas justo después de la línea def. Es lo que aparece cuando preguntas a Copilot o escribes help(funcion).

PYTHON
def calcular_ticket(facturacion: float, n_ordenes: int) -> float:
    """
    Calcula el ticket promedio dividiendo la facturación total
    por el número de órdenes distintas.

    Parámetros:
        facturacion: suma total de ventas en R$
        n_ordenes:   número de órdenes únicas

    Retorna:
        ticket promedio en R$

Valor de retorno (return)

return termina la función y devuelve un valor. Si la función no tiene return (o tiene -> None), solo ejecuta acciones sin devolver nada.

PYTHON
# Función que RETORNA un valor
def doble(n: int) -> int:
    return n * 2

resultado = doble(5)   # resultado = 10

# Función que NO retorna (solo imprime)
def saludar(nombre: str) -> None:
    print(f"Hola, {nombre}!")

saludar("Ana")   # imprime "Hola, Ana!" pero no devuelve nada

Llamar funciones
Cómo ejecutar una función definida

Llamar una función significa ejecutarla. Escribes el nombre, abres paréntesis, pasas los argumentos y cierras paréntesis.

PYTHONPatrón general
# Definición (se hace UNA vez)
def inspeccionar(df: pd.DataFrame, nombre: str) -> None:
    print(df.shape)

# Llamada (se puede hacer MUCHAS veces)
inspeccionar(pagos, "Pagos")   # primera llamada
inspeccionar(items, "Ítems")   # segunda llamada, mismo código

f-strings — la forma moderna de combinar texto y variables

Una f-string (formatted string) permite insertar variables directamente dentro del texto. Se escribe con una f antes de las comillas y las variables van entre { }.

PYTHON
nombre = "Ana"
edad   = 28

# Sin f-string (engorroso)
print("Hola, " + nombre + ". Tienes " + str(edad) + " años.")

# Con f-string (limpio y directo)
print(f"Hola, {nombre}. Tienes {edad} años.")
#      ↑ la f indica que es una f-string
#              ↑ variable insertada   ↑ otra variable
Salida (ambas líneas dan lo mismo)

Pathlib — rutas de archivos
Manejar carpetas y archivos de forma segura

¿Por qué pathlib y no strings?

En Windows las rutas usan \ y en Mac/Linux usan /. Si escribes la ruta como texto ("data/raw/archivo.csv"), tu código puede fallar en otro sistema. pathlib.Path maneja esto automáticamente — funciona igual en todos los sistemas.

PYTHON01_leer_csv.py líneas 15, 23-27, 30
from pathlib import Path

def raiz_repo() -> Path:
    """Encuentra la raíz del repo desde este archivo."""
    aqui = Path(__file__).resolve()
    #     ↑ __file__ es la ruta del script actual
    #       .resolve() la convierte en ruta absoluta
    for carpeta in [aqui, *aqui.parents]:
        if (carpeta / ".git").exists() or (carpeta / "clases").is_dir():
            return carpeta
    return Path.cwd()

# El operador / construye rutas
DATA = raiz_repo() / "data" / "raw"
#                  ↑         ↑
#       / une carpetas igual que os.path.join() pero más legible

La tabla de datos de Python

¿Qué es un DataFrame?

Un DataFrame es una tabla de datos dentro de Python — como una hoja de Excel, pero con superpoderes. Tiene filas (registros) y columnas (campos). Pandas es la librería que los crea y manipula, y es el motor del curso.

Puedes crear columnas nuevas operando sobre columnas existentes. Pandas opera sobre columnas completas de una vez (vectorización) — no necesitas un bucle.

groupby — agrupar y resumir

¿Qué es groupby?

groupby agrupa filas según los valores de una columna y calcula estadísticas por grupo. Es como una tabla dinámica de Excel, pero en una línea de Python. Por ejemplo: suma de ventas por tipo de pago, ticket promedio por categoría, etc.

PYTHON01_leer_csv.py líneas 48-53
# Agrupar pagos por tipo y calcular: conteo, suma y promedio
total_por_tipo = (
    pagos
    .groupby("payment_type")  # agrupar por esta columna
    ["payment_value"]          # sobre esta columna calcular
    .agg(["count", "sum", "mean"])  # estas estadísticas
    .sort_values("sum", ascending=False)  # ordenar por suma
    .round(2)                    # redondear a 2 decimales
)
print(total_por_tipo.to_string())

glob — automatizar con patrones
Encontrar y leer múltiples archivos de una vez

¿Qué es glob?

glob busca archivos que coincidan con un patrón. El patrón *.csv significa "todos los archivos que terminen en .csv". En vez de abrir 9 archivos a mano, una sola llamada te devuelve la lista de todos.

PYTHON02_glob_inventario.py línea 34
from glob import glob

# glob devuelve una lista de rutas que coinciden con el patrón
archivos = sorted(glob(str(DATA / "*.csv")))
#                        ↑           ↑
#                  str() convierte  *.csv = todos los CSV
#                  Path a string

print(archivos)
# ['/ruta/data/raw/olist_customers_dataset.csv',
#  '/ruta/data/raw/olist_geolocation_dataset.csv', ...]

print(f"Encontré {len(archivos)} archivos")
# Encontré 9 archivos

Bucles for
Repetir una acción para cada elemento de una lista

Un bucle for ejecuta un bloque de código una vez por cada elemento de una lista, rango o colección. Es la forma de automatizar tareas repetitivas.

PYTHON02_glob_inventario.py líneas 44-55
# Para CADA archivo en la lista de archivos:
filas_inventario = []   # lista vacía donde iremos acumulando

for ruta in archivos:
#   ↑         ↑
# variable   lista de archivos (9 en total)
# En cada iteración, "ruta" toma el valor del siguiente archivo

    df = pd.read_csv(ruta)              # leer el CSV
    nulos_pct = round(df.isna().mean().mean() * 100, 1)

    filas_inventario.append({          # agregar una fila al inventario
        "archivo": Path(ruta).name,
        "filas": df.shape[0],
        "columnas": df.shape[1],
        "memoria_MB": round(df.memory_usage(deep=True).sum() / 1e6, 1),
        "nulos_%": nulos_pct,
    })
# Después del bucle, filas_inventario tiene 9 diccionarios
# uno por cada archivo

if __name__ == "__main__"
El punto de entrada del script

¿Para qué sirve?

Esta línea le dice a Python: "ejecuta lo que sigue solo cuando este archivo se corre directamente". Si otro script importa este archivo como librería, el código dentro del if no se ejecuta automáticamente. Es una buena práctica que hace el código más modular y testeable.

PYTHON01_leer_csv.py líneas 81-82
# Definición de funciones (arriba del archivo)
def inspeccionar(df, nombre): ...
def main(): ...

# Punto de entrada — al final del archivo
if __name__ == "__main__":
    main()
# ↑ Si corres "python 01_leer_csv.py" → __name__ vale "__main__"
#   → se ejecuta main()
# ↑ Si otro script hace "import 01_leer_csv" → __name__ vale
#   el nombre del módulo → main() NO se ejecuta solo
💡
Regla de oro: todo el código ejecutable va dentro de funciones (como main()). Al final del archivo, llamas a main() dentro del if __name__ == "__main__". Esto es el patrón estándar en Python profesional.
PYTHONPatrón completo de un script profesional
# 1. Imports (siempre primero)
from pathlib import Path
import pandas as pd

# 2. Constantes
DATA = Path("data/raw")

# 3. Definición de funciones
def procesar(ruta: Path) -> pd.DataFrame:
    return pd.read_csv(ruta)

def main() -> None:
    df = procesar(DATA / "archivo.csv")
    print(df.head())

# 4. Punto de entrada
if __name__ == "__main__":
    main()
